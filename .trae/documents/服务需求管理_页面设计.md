# 服务需求管理 - 全新 UI/交互设计（不改字段与状态机）

目标：在不改变业务字段与状态机流转的前提下，提供“国际一线产品水准”的桌面端体验：强信息密度但不压迫、极少点击完成任务、对状态/风险高度可视化、每个动作都有即时反馈与可撤销。

## 1. 设计语言
### 视觉基调
- 风格：现代企业级 SaaS（类似 Linear / Stripe Dashboard 的信息架构 + Notion 的留白节奏）。
- 色彩：`zinc` 作为中性底色；主色 `indigo`；危险 `rose`；成功 `emerald`。
- 层级：弱化边框，强化阴影与半透明分隔；大面积使用卡片与 sticky header。

### 字体与排版
- 字号体系：12/13/14/16/20（仅 5 档）；正文 14，表格 13。
- 标题：页面标题 20 semibold；区块标题 16 semibold。

### 组件基准
- Button：Primary（indigo）、Secondary（zinc）、Danger（rose）、Ghost。
- Badge：按状态映射颜色与图标。
- Input：带前缀 icon；支持 clear。
- Table：sticky header + hover 行；可选行；密度切换（Comfortable/Compact）。
- Drawer/Modal：审批动作用 Drawer（右侧滑入，避免打断上下文）。
- Toast：动作反馈可撤销（撤回/删除提供 5s Undo）。

## 2. 全局信息架构
### 顶部 App Header
- 左侧：产品名 + 当前模块（“服务采购需求”）。
- 中间：全局搜索（支持键盘 `⌘K` 呼出 Command Palette）。
- 右侧：
  - “新建需求”主按钮。
  - 用户切换器（mock：切换“业务方/审批人”身份与部门）。
  - 通知入口（可选，后续接待办/提醒）。

### 左侧导航（可折叠）
- 工作台
  - 我的待办
  - 我处理的
  - 我发起的
- 管理
  - 服务需求管理

## 3. 页面设计

### 3.1 列表页（我的待办 / 我处理的 / 我发起的 / 服务需求管理）
四个列表共享同一 UI 组件，仅默认筛选口径不同。

#### 顶部区域（Sticky）
- 页面标题 + 统计概览卡（3-4 个）：
  - 待处理数量、审批中数量、已驳回数量、已完成数量。
- 视图切换 Tabs：我的待办 / 我处理的 / 我发起的 / 管理。
- 快捷筛选 Chips（可多选）：草稿 / 审批中 / 已驳回 / 已完成。

#### 高级筛选（折叠面板）
- 默认收起；点击“筛选”展开。
- 字段（不增不减，只做呈现优化）：
  - 项目名称（输入）
  - 需求编号（输入/选择，按实现 mock）
  - 需求时间（起止）
  - 地区公司（选择）
  - 部门（选择）
  - 项目类型（选择）
  - 采购方式（选择）
- 交互：
  - Enter 触发查询；Esc 关闭筛选；支持“一键清空”。
  - 筛选结果在顶部以“Active Filters”胶囊展示并可单独移除。

#### 表格（信息密度与可读性兼得）
- 行首：状态 Badge + 需求编号。
- 主列：项目名称（两行截断，hover 展示完整）
- 次列：申请单位/部门（小号文字）
- 其他列：采购方式、项目类型、需求时间、更新人/更新时间（可选）
- 操作列：
  - 主要动作固定为“查看”。
  - 行 hover 显示更多菜单（复制编号、打开流程、复制链接）。

#### 列表交互增强
- 行点击可直接进入详情（操作列仍保留“查看”链接）。
- “我的待办”行右侧展示待你处理的动作提示（如“待审批/待处理”）。
- 空态：根据视图展示引导（例如“新建需求”“清空筛选”）。

### 3.2 创建/编辑页（草稿/驳回后编辑）
#### 页面结构
- 顶部 Sticky Action Bar：
  - 左侧：返回 + 标题（新建/编辑）
  - 中间：自动保存状态（Saved/Unsaved）
  - 右侧：保存草稿、提交（主按钮）、取消。

#### 表单布局（不改变字段，仅提升节奏）
- 左侧：内容区（分组卡片）。
- 右侧：进度侧栏（字段完成度、必填缺失提示、快速跳转锚点）。

#### 字段分组（字段不增不减）
1) 基本信息
  - 资金渠道（必填，枚举：投资/成本/成本+投资）
  - 申请单位（必填）
  - 申请部门（必填）
  - 项目名称（必填）
  - 需求时间（必填，日期）
  - 采购方式（必填，枚举：直接采购/询比采购/竞价采购/谈判采购/公开招标/邀请招标）
  - 项目类型（必填，枚举：工程类/工程服务类/其他服务）
  - 公开选商（必填，枚举：是/否；存在联动规则）
  - 是否设控制价（必填，枚举：是/否）
  - 控制价（不含税，万元）（必填；按“是否设控制价”显隐）

2) 附件信息（以任务式上传体验呈现）
  - 采购依据（必填，子类枚举：立项文件/计划文件/费用文件/项目说明/预算审批意见/预算表/其他）
  - 技术方案及批复（必填，子类枚举：待定）
  - 采购需求（必填，子类枚举：招标需求表/其他）
  - 上传规则提示：支持 pdf/doc/docx/jpg/jpeg/png/xlsx/xls，单文件 ≤10MB。

3) 非公开选商信息（仅当“公开选商=否”时显示且必填项生效）
  - 供应商邀请理由及产生方式（必填，长文本）
  - 拟邀请参加选商的供应商（必填，选择/多选，mock）
  - 预算项目(CBS)（必填）
  - 资格材料（必填上传）
  - 审计报告（必填上传）

4) 采购需求信息
  - 项目概况（必填，长文本）
  - 选商范围（必填，长文本）
  - 选商要求（必填，长文本）
  - 主要技术要求（选填，长文本）
  - 实施地点（必填）
  - 服务（采购）期限（必填，日期/区间）
  - 备注（选填）
  - 标段（标包）划分及标段划分理由（选填，长文本）
  - 申请人（必填）
  - 联系方式（必填）
  - 签约主体（必填）

#### 表单交互
- 必填校验：实时但不打断（字段 blur 后提示）；提交时汇总缺失并滚动定位。
- 联动：
  - 若“公开选商=否”：展示“非公开选商信息”区块并强制必填。
  - 若“是否设控制价=否”：隐藏“控制价（不含税，万元）”。
- 上传：拖拽上传 + 进度条 + 失败重试；文件列表支持下载/移除。

### 3.3 详情页（单据/流程）
#### 顶部 Sticky Header
- 左侧：返回 + 需求编号 + 状态 Badge。
- 右侧：根据状态机与角色展示动作按钮（不越权、不可走捷径）。

#### 双栏布局
- 左：内容 Tabs
  - 单据：只读字段，以“字段卡片 + 关键字段突出”呈现（项目名称、申请单位/部门、采购方式、项目类型、控制价等）。
  - 流程：
    - 上部：流程可视化（采用“状态时间轴 + 当前节点高亮”，比静态图更可读）。
    - 下部：流转记录表（节点名称/部门/处理人/节点操作/处理意见/开始时间/结束时间）。
- 右：动作面板（Context Panel）
  - 当前你可做什么（Allowed actions）
  - 风险提示（如：驳回原因、缺失附件）
  - 审批动作入口（通过/驳回/撤回/处理/删除）

#### 动作抽屉（Drawer）
- “通过/驳回”在抽屉中完成：展示关键字段摘要 + 输入处理意见 + 确认按钮。
- “撤回/删除”提供二次确认 + Undo。

## 4. 状态机按钮控制（必须严格一致）
- 草稿：允许【业务方】提交、删除；允许编辑。
- 审批中：允许【提交人】撤回；允许【多人审批】通过/驳回。
- 已驳回：允许【业务方】处理（回到草稿后可再次提交）。
- 已完成：只读。
