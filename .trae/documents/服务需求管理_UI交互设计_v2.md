# 服务需求管理：UI/交互设计文档（V2｜不按原型图｜国际优秀案例对标）

> 覆盖页面：待办 / 我处理的 / 我发起的 / 管理列表 / 单据详情（桌面端优先）

## 1. 设计目标与原则

* 极致效率：减少视线跳转与滚动，关键动作“就地可达”，信息密度可控但不拥挤。

* 零歧义：状态/下一步/责任人始终清晰；审批动作具备强确认与可追溯反馈。

* 国际化体验：一致的栅格、字阶、动效节奏、空态/骨架屏、可访问性（对比度 AA）。

## 2. 信息架构（IA）

**全局壳（App Shell）**

* 左侧侧边栏（一级）：工作台（待办、我处理的、我发起的）、管理（管理列表）。

* 顶部栏：全局搜索（按“项目名称/需求编号”）、用户入口（头像/退出不在范围可隐藏）。

* 主内容区：页面标题 + 视图说明（可选）+ 筛选区 + 列表/详情内容。

**导航规则**

* 列表 -> 详情：同窗跳转（保留列表筛选条件与分页位置）。

* 详情顶部提供“返回列表”（返回到进入前的视图与滚动位置）。

## 3. 视觉规范（Design Tokens）

**布局与栅格（Desktop-first）**

* 内容最大宽度：1200–1360px；左右留白随屏宽自适应；核心列表区域优先占满。

* 12 列栅格；关键表格列保持可读宽度（项目名称列可弹性）。

**色彩**

* Brand/Primary：#2563EB（按钮/高亮/链接）

* Text：#0F172A / 次级 #475569；Border：#E2E8F0；Bg：#F8FAFC

* 语义色：成功 #16A34A｜警示 #D97706｜危险 #DC2626｜信息 #2563EB

**字阶**

* H1 20/28（页标题）；H2 16/24（区块标题）；Body 14/22；Meta 12/18。

**圆角/阴影/动效**

* Radius：8（卡片/弹窗）、6（输入/按钮）；

* Shadow：列表 hover 轻阴影；

* Motion：120ms（hover/focus），200ms（弹窗/抽屉），easing: cubic-bezier(0.2,0,0,1)。

## 4. 组件规范（关键可复用）

1. **SideNav（侧边栏）**：图标 + 文案；当前项高亮；支持折叠（保留图标）。
2. **TopBar（顶部栏）**：全局搜索输入（Enter 触发）；右侧显示当前用户（可选）。
3. **Tabs（页签）**：用于“待办/我处理的/我发起的”（在管理列表中也可复用）；显示计数（可选）。
4. **Filter Bar（条件查询区）**：

   * 组件：输入框（项目名称/编号）、日期范围、下拉（地区公司/部门/项目类型/采购方式）、重置/查询。

   * 已生效条件以“筛选Chip”展示，可单个删除。
5. **Data Table（结果列表）**：

   * 行高 44；首列为“项目名称+需求编号（副文案）”；右侧固定“状态/需求时间/查看”。

   * 行 hover：整行浅底色 + 右侧浮现“查看”按钮。
6. **Status Pill（状态标签）**：草稿/审批中/已驳回/已完成等（语义色+浅底）。
7. **Pagination（分页）**：上一页/下一页 + 页码；支持显示总数（可选）。
8. **Action Dialog（动作弹窗）**：用于通过/驳回确认；驳回需填写“处理意见”（必填）。
9. **Toast（轻提示）**：动作成功/失败反馈；失败提供“重试/查看原因”。
10. **Detail Tabs（详情页签）**：单据 / 流程；粘性（Sticky）在滚动时保持可见。
11. **Timeline Table（流转记录表）**：对齐列：节点名称/部门/处理人/节点操作/处理意见/开始/结束。

## 5. 关键页面线框与交互说明

### 5.1 管理列表（容器页）

**Meta**：Title「服务需求｜管理列表」；Description「筛选并查看服务需求单据」。

**线框（桌面）**

```
┌SideNav┐ ┌──────────────────────── TopBar(搜索) ────────────────────────┐
│ 待办  │ │ 标题：服务需求                                                  │
│ 我处理│ │ Tabs：待办 | 我处理的 | 我发起的                                 │
│ 我发起│ │ Filter：项目名称/编号 | 时间范围 | 地区公司 | 部门 | 项目类型 | 采购方式 │
│ 管理  │ │ ───────────────────────── Table ───────────────────────────── │
└───────┘ │ 行：项目名称 + 编号 | 地区公司 | 采购方式 | 需求时间 | 状态 | [查看] │
          │ …                                                               │
          │ Pagination                                                      │
          └─────────────────────────────────────────────────────────────────┘
```

**交互**

* 查询：点击“查询”刷新表格；加载中显示骨架行（3–8 行）。

* 筛选Chip：展示在筛选区下方一行；点击“x”移除单个条件并刷新。

* 查看：点击行或“查看”进入单据详情；返回列表保持原筛选与分页。

### 5.2 待办（列表视图）

**Meta**：Title「服务需求｜待办」；Description「你需要处理的审批任务」。

**视图规则**

* 默认口径：仅展示“待你审批”的记录（状态通常为审批中且当前节点=你）。

**交互强化（不改变业务能力）**

* 视觉强调：状态 Pill 使用更高对比；行首增加“待你处理”标识（弱强调，不抢主信息）。

* 键盘效率：

  * ↑/↓ 移动行焦点；Enter 打开详情；Esc 取消焦点。

### 5.3 我处理的（列表视图）

**Meta**：Title「服务需求｜我处理的」；Description「你已处理过的审批记录」。

**视图规则**

* 展示你曾执行过“通过/驳回”的记录。

* 列表中状态保持真实单据状态；在状态旁以小字补充“你的动作：通过/驳回”（仅展示，不新增操作）。

### 5.4 我发起的（列表视图）

**Meta**：Title「服务需求｜我发起的」；Description「你创建并提交的服务需求」。

**视图规则**

* 展示你作为提交人创建的记录（草稿/审批中/已驳回/已完成）。

* 草稿/已驳回行：状态 Pill 更醒目，提醒“需要处理后再提交”（仅提示）。

### 5.5 单据详情（单据/流程）

**Meta**：Title「服务需求｜详情｜{需求编号}」；Description「查看单据字段、附件与审批流程」。

**布局**

* 顶部 Header（粘性）：返回列表｜需求编号（可复制）｜状态 Pill｜右侧动作区。

* Tabs：单据 / 流程（粘性）。

* 内容区：卡片分组，减少长表单疲劳。

**线框**

```
Header: ←返回 | 编号 SR-XXXX (复制) | 状态[审批中]                [通过] [驳回] [取消]
Tabs:  单据 | 流程

[单据]
- 基本信息卡片（只读字段，2列栅格）
- 附件卡片（按类别分组：文件名｜大小｜上传时间｜下载）

[流程]
- 流程图区域（居中，自适应宽；加载失败给出重试）
- 流转记录表（Timeline Table）
```

**动作与权限（按角色呈现）**

* 审批人：展示「通过 / 驳回 / 取消」。

* 提交人：仅在“审批中”展示「撤回」（若产品已定义）；否则不显示审批按钮。

**关键交互**

* 通过：点击后弹出确认 Dialog（可选填写处理意见）；确认后 Toast 提示成功并返回上一列表。

* 驳回：弹出 Dialog；“处理意见”必填；提交后 Toast；返回列表。

* 取消：关闭弹窗或返回列表，不产生更改。

* 错误态：动作失败时保留输入内容，提示原因（如权限/网络），提供“重试”。

## 6. 通用状态与可访问性

* 空态：说明原因 + 建议（如“调整筛选条件”）；待办空态突出“暂无需要你处理的审批”。

* 加载：列表骨架屏；详情用分区骨架（Header/卡片/表格）。

* 可访问性：所有可点击元素提供 focus ring；表格行与按钮可键盘操作；颜色不作为唯一信息承载（状态同时有文案）。
